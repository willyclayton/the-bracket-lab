# The Bracket Lab

**March Madness 2026 AI Prediction Platform**

Live: [bracketlabs.vercel.app](https://bracketlabs.vercel.app)

## **Purpose**

March Madness is one of the most unpredictable events in sports — which makes it the perfect stress test for AI. The Bracket Lab pits 8 AI models against each other, each using a fundamentally different prediction strategy, with live accuracy tracking against real tournament results. The site doubles as a content platform with methodology deep-dives, round recaps, and model "obituaries" when a predicted champion gets eliminated.

What started as a conversation with a friend turned into a shipped product in under two weeks. Multiple people used the models' picks to help build — and win — their bracket pools.

## **Features**

- **8 competing AI models** — Monte Carlo simulation, archetype matching, upset detection, autonomous AI research, expected-value optimization, logistic regression, data-saturated LLM analysis, and classic film-room scouting
- **Live leaderboard** — real-time ESPN-style scoring (10/20/40/80/160/320 pts per round) recalculated client-side on every page load
- **Bracket explorer** — interactive bracket visualization with model tabs, matchup popovers, and confidence ratings per pick
- **Methodology blog** — MDX-powered deep-dives into how each model works, written for a general audience
- **Tournament recaps** — round-by-round analysis posts published during the tournament
- **Model obituaries** — dramatic eulogies published when a model's predicted champion is eliminated
- **Visitor voting** — "pick a model to root for" widget with seeded counts and bar chart reveal
- **Cheat sheet** — condensed view for quick bracket-filling reference
- **Year toggle** — browse 2024, 2025, and 2026 brackets with archived + verified historical data

## **Data Architecture**

All data lives in `data/*.json` — no external database. Model predictions are generated by Python pipelines and committed as static JSON. Tournament results are manually updated from ESPN/NCAA.com with source URLs in commit messages.

- **Model bracket data** (`data/models/`) — 9 JSON files, one per model, containing every pick with confidence scores and reasoning
- **Actual results** (`data/results/actual-results.json`) — real game outcomes updated during the tournament
- **Team metadata** (`data/meta/teams.json`) — seeds, regions, conferences for the current bracket
- **Research data** (`data/research/`) — historical team archetypes, upset factors, seed performance history
- **Archives** (`data/archive/2024/`, `data/archive/2025/`) — fully verified historical brackets and results

Data integrity is non-negotiable: model predictions are AI-generated (that's the point), but every game score and bracket structure is verified against official sources.

## **Architecture Decisions**

**Static JSON over a database.** Tournament data changes at most a few times per day during game windows. Serving pre-built JSON from the CDN is faster, cheaper, and simpler. The tradeoff is manual result updates, which is acceptable for data integrity — every score gets human-verified before commit.

**Client-side scoring engine.** Scores are recalculated in the browser on every page load by comparing model picks against actual results. No server-side computation needed. When results are updated and Vercel redeploys (~30 seconds), every visitor sees fresh scores immediately.

**One model metadata file to rule them all.** `lib/models.ts` is the single source of truth for names, colors, taglines, slugs, and scoring constants. Components never hardcode model-specific values — everything flows from this file.

**MDX for content.** Blog posts and methodology writeups are MDX files with frontmatter metadata. This keeps content in the repo, version-controlled, and deployable with zero CMS overhead.

## **Learnings**

**Process.** Do heavy research before writing code. UI iteration never ends — what looks right on the first pass rarely survives real user feedback. Once you go live, bugs need to be fixed immediately because people are watching and a broken experience stays broken until you fix it.

**Technical.** Stripe integrations, live scoring pipelines, Redis caching, MDX blogging, and keeping interactions simple and the site clean. Using AI as a researcher to surface data points I wouldn't have thought to look for. AI "looping" — iterating with AI tools to converge on the best approach rather than accepting the first output.

**Marketing.** Tried using social media to drive traffic. The real lesson: the product has to be interesting enough to share on its own.

**The fun part.** March Madness is awesome precisely because it's unpredictable — and multiple people actually used this to help win their bracket pools, which was the whole point.
