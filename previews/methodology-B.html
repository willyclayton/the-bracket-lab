Quick Mental Model
Core Philosophy

The Scout treats every matchup the way a real analyst would. Where efficiency models see numbers, it sees context: the injured point guard, the road-weary roster, the first-year coach about to crack.

Strengths
  • Captures intangibles pure efficiency models miss
  • Contextual synthesis — 6 factors weighted simultaneously
  • No normalization required — model understands magnitude natively
Key Inputs
  • ·Coaching experience — tournament record
  • ·Clutch performance — games decided by ≤5 pts
  • ·Roster age — senior vs. freshman distribution
  • ·Momentum — last 10 games, conference tournament
  • ·Injury / availability — beat reporters + official logs
  • ·Travel and rest — distance, days between games

    Methodology
    The Actual Build
    The Process
    01
    Pull team dataCoaching record, roster age breakdown, injury log, last-10 results, travel schedule for each team.
    02
    Build matchup context objectsPair Team A vs. Team B data into a structured JSON context block for all 32 first-round games.
    03
    Run structured analyst promptFeed each context block to Claude with analyst role. Returns scouting report + confidence-weighted pick in JSON.
    04
    Advance bracket round by roundWinners from Round 1 become the context for Round 2 matchups. Repeat through the championship.
    05
    Lock picks March 19Final bracket exported to JSON. ESPN Tournament Challenge entry submitted as timestamp proof.
    Data Sources
    • KenPom — efficiency ratings (2026 season through conference tournaments)
    • Sports Reference — coaching records and tournament history
    • Beat reporter injury coverage (ESPN, team writers)
    • Official injury designations where available
    Key Assumptions
    • LLM contextual synthesis produces better picks than regression on the same inputs
    • Coaching experience is a consistent predictor across eras
    • Injury data 48 hours before picks lock is sufficiently current
    Limitations
    • Hallucination risk on specific stats — mitigated by injecting verified data
    • LLM tendency to favor brand-name programs — explicitly prompted against, imperfectly corrected
    • No live updates after March 19
